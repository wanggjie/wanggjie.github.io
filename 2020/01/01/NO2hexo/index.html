从 0.0 开始搭建自己的个人博客 Hexo

1：安装node环境

下载地址node.js地址：https://nodejs.org/en

下载 → 双击安装 → 无脑继续/下一步

查看是否安装成功 → 打开终端（win+R）输入命令 cmd 出现终端窗口 → 输入node -v 查看版本号信息

如果下面显示版本号就已经安装成功了

（因为 hexo 需要 node.js 来生成的，所以前置工作要 node.js 支持，否则无法继续下去）

2：安装全局 Hexo

Hexo 官网地址：https://hexo.io/zh-cn/

可以在终端窗口继续操作输入 npm install hexo-cli -g 命令安装全局 Hexo

检验是否安装成功 → 输入命令 hexo -v 查看版本信息 → 安装成功

3：创建Hexo

创建一个空文件夹 → 如果途中出现什么问题，可以直接删掉重来

空文件夹开启终端窗口 → 输入命令（hexo init）初始化 Hexo → 最后显示 Start blogging with Hexo! 文字表示初始化成功 → 可以看到文件夹下面已经生成了很多 hexo 基础的一些文件夹/文件

启动 hexo 博客 → 输入命令 hexo s → 显示在本地打开地址：http://localhost:4000/

可以试着创建一篇博客文章 → 跟着网站下面的一些命令操作 → hexo new “My New Post（文章标题）” 创建新文章 → 创建完成可以再 source 文件下面的 _post 文件夹看到相对于的 md 文档 → hexo generate 生成在hexo博客上 → 再启动hexo → 生成完成

目前到这里的话，hexo已经搭建完成了

接下来我们把它部署到 GitHub 远端上面去，不仅仅止于在本地访问

4：部署到远端 GitHub

打开 GitHub 官网 → 创建新的仓库（用户部署个人博客的 GitHub 仓库命名必须符合特定要求才行）【GitHub的昵称.github.io】 → 后续通过这个名称访问自己的博客

创建新仓库完成后 → 需要在hexo文件夹下创建一个git部署的插件 npm install –save hexo-deployer-git

通过npm下载完成后 → 需要设置 _config.yml 文件 → 打开文件在最后面修改/添加

修改保存后 → 部署到GitHub上面去 → 输入命令 hexo d → 完成绑定 GitHub 账号密码 → 推送完成（刷新GitHub仓库便可查看到已上传的文件）

然后在通过仓库名可以远程访问到自己的博客了（ wanggjie.github.io ）

目前到这hexo已经搭建完毕
